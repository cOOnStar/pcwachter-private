# Datenbank – Schema

## Technologie

PostgreSQL 17 – Source of Truth für alle Geschäftsdaten.
Migrationen via Alembic (`server/api/alembic/versions/`).

## Tabellen

### devices

| Spalte | Typ | Beschreibung |
|---|---|---|
| `device_install_id` | VARCHAR PK | Eindeutige Install-ID des Geräts |
| `keycloak_user_id` | VARCHAR | Zugehöriger Keycloak-User |
| `hostname` | VARCHAR | Gerätename |
| `os_version` | VARCHAR | Windows-Version |
| `agent_version` | VARCHAR | Agent-Version |
| `last_seen` | TIMESTAMPTZ | Letzter Heartbeat |
| `is_online` | BOOLEAN | Abgeleitet aus `last_seen` + `ONLINE_THRESHOLD_SECONDS` |
| `created_at` | TIMESTAMPTZ | Registrierungszeitpunkt |

### device_tokens

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Token-ID |
| `device_install_id` | VARCHAR FK → devices | Zugehöriges Gerät |
| `token_hash` | VARCHAR | SHA-256 Hash des Token |
| `expires_at` | TIMESTAMPTZ | Ablaufzeitpunkt (NULL = unbefristet) |
| `created_at` | TIMESTAMPTZ | Ausstellungszeitpunkt |
| `revoked_at` | TIMESTAMPTZ | Widerrufszeitpunkt (NULL = aktiv) |

### telemetry_snapshots

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Snapshot-ID |
| `device_install_id` | VARCHAR FK → devices | Zugehöriges Gerät |
| `payload` | JSONB | Rohdaten (CPU, RAM, Disk, Antivirus, ...) |
| `captured_at` | TIMESTAMPTZ | Aufnahmezeitpunkt |
| `created_at` | TIMESTAMPTZ | Eingangszeit |

### update_results

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Ergebnis-ID |
| `device_install_id` | VARCHAR FK → devices | Zugehöriges Gerät |
| `status` | VARCHAR | `success` / `failed` / `pending` |
| `details` | JSONB | Detaildaten (Fehlertext, Paketliste) |
| `created_at` | TIMESTAMPTZ | Zeitpunkt |

### inventory_snapshots

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Snapshot-ID |
| `device_install_id` | VARCHAR FK → devices | Zugehöriges Gerät |
| `payload` | JSONB | Inventardaten (installierte Software, etc.) |
| `created_at` | TIMESTAMPTZ | Eingangszeit |

### licenses

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Lizenz-ID |
| `license_key` | VARCHAR UNIQUE | Aktivierungsschlüssel |
| `keycloak_user_id` | VARCHAR | Zugehöriger User |
| `plan_id` | VARCHAR FK → plans | Zugehöriger Plan |
| `device_install_id` | VARCHAR | Gebundenes Gerät |
| `status` | VARCHAR | `active` / `revoked` / `expired` |
| `valid_from` | TIMESTAMPTZ | Gültigkeitsbeginn |
| `valid_until` | TIMESTAMPTZ | Ablaufdatum (NULL = unbefristet) |
| `created_at` | TIMESTAMPTZ | Ausstellungszeitpunkt |

### plans

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | VARCHAR PK | Plan-ID (`trial`, `standard`, `professional`, `unlimited`, `custom`) |
| `name` | VARCHAR | Anzeigename |
| `price_monthly` | NUMERIC | Monatspreis |
| `max_devices` | INTEGER | Max. Geräte (NULL = unbegrenzt) |
| `duration_days` | INTEGER | Laufzeit in Tagen (NULL = unbefristet) |
| `feature_flags` | JSONB | Plan-Level Feature-Flags |
| `grace_period_days` | INTEGER | Kulanzzeit nach Ablauf |
| `stripe_price_id` | VARCHAR | Stripe Price ID |
| `is_active` | BOOLEAN | Buchbar/sichtbar |

### subscriptions

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Subscription-ID |
| `keycloak_user_id` | VARCHAR | Zugehöriger User |
| `license_id` | UUID FK → licenses | Verknüpfte Lizenz |
| `plan_id` | VARCHAR FK → plans | Gebuchter Plan |
| `status` | VARCHAR | `active` / `canceled` / `past_due` |
| `stripe_customer_id` | VARCHAR | Stripe Customer |
| `stripe_subscription_id` | VARCHAR | Stripe Subscription |
| `stripe_price_id` | VARCHAR | Stripe Price |
| `current_period_start` | TIMESTAMPTZ | Abrechnungsperiode Beginn |
| `current_period_end` | TIMESTAMPTZ | Abrechnungsperiode Ende |
| `grace_until` | TIMESTAMPTZ | Kulanzfrist |
| `trial_used` | BOOLEAN | Trial bereits genutzt |
| `created_at` | TIMESTAMPTZ | Erstellungszeitpunkt |
| `updated_at` | TIMESTAMPTZ | Letzte Änderung |

### feature_overrides

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Override-ID |
| `feature_key` | VARCHAR | Feature-Schlüssel |
| `enabled` | BOOLEAN | Erzwungener Status (Kill-Switch) |
| `rollout_pct` | INTEGER | Rollout-Prozentsatz (0-100) |
| `version_min` | VARCHAR | Min. Agent-Version |
| `platform` | VARCHAR | Plattform-Gate (`windows`, `all`) |
| `notes` | TEXT | Admin-Notiz |
| `updated_at` | TIMESTAMPTZ | Letzte Änderung |

### webhook_events

| Spalte | Typ | Beschreibung |
|---|---|---|
| `id` | UUID PK | Event-ID |
| `event_type` | VARCHAR | Stripe Event-Typ (z.B. `customer.subscription.updated`) |
| `stripe_event_id` | VARCHAR UNIQUE | Stripe Event-ID (Idempotenz) |
| `payload` | JSONB | Vollständiger Stripe Payload |
| `processed_at` | TIMESTAMPTZ | Verarbeitungszeitpunkt |
| `created_at` | TIMESTAMPTZ | Eingangszeit |

## Feature-Flag-Logik

```
Final Features = plan_flags AND override.enabled AND rollout_pct AND version_min AND platform
```

1. `plans.feature_flags` (JSONB) → Darf grundsätzlich (Plan-Level)
2. `feature_overrides` → Kill-Switch / stufenweiser Rollout (Ops-Level)
